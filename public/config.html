Data Providers Guide & Realistic Examples

Data Providers (or Southbound Connectors) are the bridges that funnel external telemetry into Korelate's internal Unified Namespace (UNS). You can configure as many providers as you need, mixing different industrial protocols.

1. Universal Security & Routing Fields

  • Provider ID: A unique technical name for internal routing (e.g., factory_emqx_main or bms_opcua_01). Do not use spaces.
  • Subscribe Topics: A comma-separated list defining exactly what data to ingest. Syntax changes completely depending on the chosen protocol (see below)!
  • Publish Allowed Topics: A security sandbox. List the topics this connector is authorized to write back to (e.g., for sending commands to PLCs). Leave empty to enforce strict Read-Only access.

2. Protocol-Specific Configurations

MQTT Broker (Standard IoT)

Connects to standard brokers like Mosquitto, EMQX, or AWS IoT.

  • Host & Port: broker.hivemq.com / 1883 (TCP) or 8883 (TLS).
  • Subscribe: Uses MQTT wildcards. Example: enterprise/site/area/# (all subtopics), or factory/+/temp (single level).
  • Security: Upload Certificates (.crt/.key) if MTLS is required by your IT department.
Modbus TCP (Industrial)

Legacy protocol for PLCs and sensors.

  • Unit ID: Slave ID (usually 1).
  • Subscribe: Use Addr:Len::Topic. Example: 40001:1::factory/temp (Holding Register 1).
Siemens S7 (S7-Comm)

Native connection to Siemens S7-300/1200/1500.

  • Rack/Slot: Usually 0/1 or 0/2.
  • Subscribe: Use Tag::Topic. Example: DB1,REAL4::factory/pressure.
EtherNet/IP (CIP)

Rockwell (Allen-Bradley) and Omron automation.

  • Routing: JSON array like [1, 0] (Port 1, Slot 0).
  • Subscribe: Use CIP_Tag::Topic. Example: MyTag[0]::factory/vibration.
OPC UA Server (Industrial)

Connects directly to PLCs or SCADA gateways (e.g., Kepware, Ignition).

  • Endpoint URL: opc.tcp://10.0.0.55:4840.
  • Subscribe: You MUST provide exact NodeIDs, not topics! Example: ns=1;s=Temperature, ns=2;i=1045.
  • Behavior: Korelate automatically subscribes, polls data at the sampling interval, and wraps the value, quality, and timestamp in JSON.
BACnet/IP (Building Automation)

Connects to BMS systems, HVAC, and smart building controllers.

  • Target IP: IP of the specific BACnet device (e.g. 192.168.1.15).
  • Subscribe: Use Type:Instance:Property format. Example: 0:1:85 (Analog Input 1, Present Value).
KNX/IP (Building Automation)

Event-driven standard for commercial building lighting, blinds, and HVAC.

  • Gateway IP: IP of the KNX IP Router/Interface.
  • Subscribe: Use GroupAddress:DPT format. Example: 1/1/1:DPT1.001 (Switch On/Off).
  • Behavior: Listens directly to GroupValueWrite events on the bus instead of polling.
I3X Server (RFC 001)

Connects to another compliant Unified Namespace (like another Korelate instance) via REST/SSE.

  • Base URL: http://remote-korelate:8080/api/i3x.
  • Subscribe: Use Element IDs defined in the remote CDM. Example: Pump-001, CNC-Machine-5.
  • Behavior: Establishes a persistent Server-Sent Events (SSE) stream for real-time contextual data.
HTTP Webhook (Ingest API)

Opens a REST endpoint for systems that can only push HTTP POST requests (e.g., ERPs, old sensors).

  • Path Prefix: /api/ingest/erp.
  • Usage: The remote system must POST to: http://<korelate-ip>:8080/api/ingest/erp/my/custom/topic with a JSON body.
SQL Database Poller

Periodically executes a SELECT query on PostgreSQL, MySQL, or MS SQL Server to stream new rows.

  • Cursor Column: E.g., updated_at. Used to ensure only new data is fetched on subsequent polls.
  • Topic Column: E.g., asset_id. Dynamically appends to the Base Topic (e.g., base/topic/{asset_id}).
REST API Poller

Actively interrogates HTTP GET endpoints at a given interval.

  • Auth: Supports Basic Auth, Bearer Token, or API Keys.
  • Behavior: Converts the JSON response into a payload and forwards it to the defined Topic.
SNMP Poller

Polls network equipment (routers, switches) using specific OIDs.

  • OIDs: Comma-separated list of Object Identifiers (e.g., 1.3.6.1.2.1.1.1.0).
  • Community: Usually public. Supports SNMP v1 and v2c.
Apache Kafka

High-throughput integration with enterprise Kafka clusters.

  • Brokers: E.g., kafka1:9092, kafka2:9092.
  • Subscribe Topics: Topics to consume from.
  • Publish Topics: Allowed topics for producing messages.
Local File Stream (Simulation/Playback)

Reads a CSV or JSONL file from disk and streams it line-by-line into the UNS as if it were live data. Perfect for testing AI models or UI dashboards without real hardware.

  • File Path: data/test_data.csv.
  • Stream Rate: 1000 (emits one row every 1000ms).
  • Routing: If your CSV has a column named topic, Korelate uses it. Otherwise, it uses the fallback topic you provide.

Note: For OPC UA, 'Subscribe Topics' should be NodeIDs (e.g. ns=1;s=Temperature).

Note: For Modbus, 'Subscribe Topics' should map Addresses to UNS Topics: Address:Length::UNS/Topic (e.g. 40001:16::factory/temp).

Note: For S7-Comm, 'Subscribe Topics' should map S7 Tags to UNS Topics: S7_Tag::UNS/Topic (e.g. DB1,REAL4::factory/pressure).

Note: For EtherNet/IP, 'Subscribe Topics' should map CIP Tags to UNS Topics: CIP_Tag::UNS/Topic (e.g. MyArray[0]::factory/flow).

Note: For BACnet, 'Subscribe Topics' should map Type:Instance:Property to UNS Topics (e.g. 0:1:85::bms/room1/temp for AnalogInput 1 PresentValue).

Note: KNX is event-driven. 'Subscribe Topics' should map GroupAddress:DPT to UNS Topics (e.g. 1/1/1:DPT1.001::bms/light/1).

Note: For I3X, 'Subscribe Topics' should be Element IDs (e.g. Pump-001).
