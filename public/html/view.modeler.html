📚 CDM & UNS Modeler Guide

The Common Data Model (CDM) Modeler allows you to structure your raw data streams into a contextualized Unified Namespace (UNS). This model is 100% compliant with the I3X (RFC 001) standard, making it fully interoperable with compliant edge and cloud systems.

Key Concepts

  • Object Types: The blueprints or classes for your logical entities (e.g., PumpType, HVAC_Unit, Microservice). They define the expected attributes.
  • Instances (Objects): The actual physical or logical entities in your system (e.g., Pump-001). They inherit from an Object Type and are mapped to real-time streams from your Data Providers (MQTT, OPC-UA, HTTP, APIs).
  • Relationships: Links connecting instances together to form your digital twin graph (e.g., Pump-001 → HasComponent → Motor-A).

Agnostic Hierarchies

Korelate is highly flexible and domain-agnostic. The "Hierarchy Level" field is entirely optional and depends on your industry context:
• Manufacturing / IT-OT: Use the ISA-95 standard (Enterprise → Site → Area → Line → Cell).
• Smart Buildings (BMS): Use Brick Schema or standard topologies (Building → Floor → Room → Zone).
• Other Domains: Leave it blank or define your own custom levels.
